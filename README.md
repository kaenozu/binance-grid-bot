# Binance グリッド取引ボット

Binance API を使った自動グリッド取引ボットです。価格が一定の範囲で動く相場（レンジ相場）で効果を発揮し、小幅な値動きを繰り返すことで利益を積み上げます。

## 免責事項

- 本ツールは利益を保証するものではありません
- 仮想通貨取引には元本割れなどのリスクが伴います
- 必ず Binance Testnet で動作確認してから本番環境で使用してください
- 実際の資金を使用した取引は、すべて自己責任となります

## 特徴

- **自動グリッド取引**：設定した価格帯に買い注文と売り注文を自動配置
- **リスク管理**：損切りラインの自動発動、最大ポジション数の制限
- **リアルタイム監視**：ターミナルへ定期的にステータスを表示
- **Testnet 対応**：Binance のテストネットワークで安全に動作確認可能
- **詳細ログ**：全取引履歴をファイルに記録
- **損益管理**：実現利益と未実現利益を自動計算

## セットアップ

### 1. 依存ライブラリのインストール

```bash
pip install -r requirements.txt
```

### 2. Binance API キーの用意

**Testnet 用（推奨・無料）**

1. https://testnet.binance.vision/ にアクセス
2. GitHub アカウントでログイン
3. API キーを生成

**本番用**

1. Binance にログイン
2. API 管理画面でキーを作成
3. 「スポット取引」の権限を有効化
4. IP アドレス制限を推奨

### 3. 環境変数の設定

```bash
copy .env.example .env
```

`.env` ファイルをエディタで開き、以下を設定してください。

```env
BINANCE_API_KEY=あなたのAPIキー
BINANCE_API_SECRET=あなたのAPIシークレット

TRADING_SYMBOL=BTCUSDT
GRID_COUNT=30
INVESTMENT_AMOUNT=500
STOP_LOSS_PERCENTAGE=5
MAX_POSITIONS=5

USE_TESTNET=true
```

## 使用方法

### 起動

```bash
python main.py
```

### 停止

`Ctrl+C` で停止します。配置済みの注文は自動的にキャンセルされます。

## 設定項目

| 項目 | 説明 | 初期値 | 備考 |
|------|------|--------|------|
| `TRADING_SYMBOL` | 取引ペア | BTCUSDT | Binance で取引可能なペア |
| `GRID_COUNT` | グリッド数 | 30 | 2 以上を推奨 |
| `INVESTMENT_AMOUNT` | 総投資額（USDT） | 500 | 0 より大きい値 |
| `STOP_LOSS_PERCENTAGE` | 損切りライン（%） | 5 | 0 〜 100 |
| `MAX_POSITIONS` | 同時保有ポジション数 | 5 | 1 以上 |
| `USE_TESTNET` | Testnet 利用設定 | true | 本番は false |

`LOWER_PRICE` と `UPPER_PRICE` は未設定の場合、現在価格の ±15% が自動設定されます。

### グリッド数の目安

| グリッド数 | 特徴 |
|-----------|------|
| 少ない（10 前後） | 1 回あたりの利益は大きいが、取引機会が減る |
| 多い（30 〜 50） | 取引機会が増えるが、1 回あたりの利益は小さくなる |

## グリッド取引の仕組み

価格帯を等間隔に分割し、各レベルに買い注文と売り注文を置きます。

```
価格範囲: $60,000 〜 $80,000
グリッド数: 10
間隔: $2,000

$80,000 ─── 売り注文
$78,000 ─── 売り注文
$76,000 ─── 売り注文
  ...
$62,000 ─── 買い注文
$60,000 ─── 買い注文
```

価格が $70,000 から $68,000 に下がると買い注文が約定し、$68,000 から $70,000 に戻ると売り注文が約定します。この往復 1 回で「グリッド間隔 × 数量」分の利益が入ります。

### 得意な相場

- **レンジ相場**（一定範囲で上下）：利益機会が増える
- **トレンド相場**（一方向に大きく動く）：含み損が膨らむ可能性がある

## リスク管理

### 損切り

開始時の価格から設定割合（デフォルト 5%）下落すると、すべての注文をキャンセルして停止します。

### ポジション制限

同時に保有できるポジション数を制限します（デフォルト：5）。上限に達すると新規の買い注文を出しません。

### 自動リトライ

API エラーやネットワーク切断時、最大 3 回まで自動的に再試行します。

## 注意事項

1. 最初は必ず Testnet でテストしてください
2. 余剰資金の範囲内で使用してください
3. 定期的にログファイルを確認してください
4. 安定したネットワーク環境で実行してください
5. API キーは他人と共有しないでください
6. 本番環境では少額から始めてください

## トラブルシューティング

### 「設定エラーがあります」と表示される

`.env` ファイルの `BINANCE_API_KEY` と `BINANCE_API_SECRET` が正しく設定されているか確認してください。

### 「API リクエストに失敗しました」と表示される

- API キーに「スポット取引」の権限があるか確認
- IP ホワイトリストが設定されているか確認
- Testnet キーと本番キーを混同していないか確認

### 注文が約定しない

- グリッド間隔が広すぎる可能性があります。グリッド数を増やしてください
- 現在価格がグリッド範囲から外れている可能性があります

### ログの確認方法

`logs/grid_bot_YYYYMMDD.log` にすべてのログが記録されています。
