# Binance グリッド取引ボット

Binance APIを使用した自動グリッド取引ボットです。価格が一定の範囲で動く相場（レンジ相場）で効果を発揮し、小幅な値動きを繰り返すことで利益を積み上げます。

## 免責事項

- 本ツールは利益を保証するものではありません
- 仮想通貨取引には元本割れなどのリスクが伴います
- 必ずBinance Testnetで動作確認してから本番環境で使用してください
- 実際の資金を使用した取引は、すべて自己責任となります

## 特徴

- **自動グリッド取引**: 設定した価格帯に自動的に買い注文と売り注文を配置
- **リスク管理**: 損切りラインの自動発動、最大ポジション数の制限
- **リアルタイム監視**: ターミナルに定期的にステータスを表示
- **Testnet対応**: Binanceのテストネットワークで安全に動作確認可能
- **詳細ログ**: 全取引履歴をファイルに記録
- **損益管理**: 実現利益と未実現利益を自動計算

## セットアップ

### 1. 依存ライブラリのインストール

```bash
pip install -r requirements.txt
```

### 2. Binance APIキーの用意

**Testnet用（推奨・無料）**
1. https://testnet.binance.vision/ にアクセス
2. GitHubアカウントでログイン
3. APIキーを生成

**本番用**
1. Binanceにログイン
2. API管理画面でキーを作成
3. 「スポット取引」の権限を有効化
4. IPアドレス制限を推奨

### 3. 環境変数の設定

```bash
copy .env.example .env
```

`.env` ファイルをエディタで開き、以下の項目を設定してください。

```env
BINANCE_API_KEY=あなたのAPIキー
BINANCE_API_SECRET=あなたのAPIシークレット

TRADING_SYMBOL=BTCUSDT
GRID_COUNT=30
INVESTMENT_AMOUNT=500
STOP_LOSS_PERCENTAGE=5
MAX_POSITIONS=5

USE_TESTNET=true
```

## 使用方法

### 起動

```bash
python main.py
```

### 停止

`Ctrl+C` で停止します。配置済みの注文は自動的にキャンセルされます。

## 設定項目

| 項目 | 説明 | 初期値 | 備考 |
|------|------|--------|------|
| `TRADING_SYMBOL` | 取引ペア | BTCUSDT | Binanceで取引可能なペア |
| `GRID_COUNT` | グリッド数 | 30 | 2以上を推奨 |
| `INVESTMENT_AMOUNT` | 総投資額（USDT） | 500 | 0より大きい値 |
| `STOP_LOSS_PERCENTAGE` | 損切りライン（%） | 5 | 0〜100 |
| `MAX_POSITIONS` | 同時保有ポジション数 | 5 | 1以上 |
| `USE_TESTNET` | Testnet使用フラグ | true | 本番はfalse |

`LOWER_PRICE` と `UPPER_PRICE` は未設定の場合、現在価格の±15%が自動設定されます。

### グリッド数の目安

| グリッド数 | 特徴 |
|-----------|------|
| 少ない（10前後） | 1回あたりの利益は大きいが、取引機会が減る |
| 多い（30〜50） | 取引機会が増えるが、1回あたりの利益は小さくなる |

## グリッド取引の仕組み

価格帯を等間隔に分割し、各レベルに買い注文と売り注文を置きます。

```
価格範囲: $60,000 〜 $80,000
グリッド数: 10
間隔: $2,000

$80,000 ─── 売り注文
$78,000 ─── 売り注文
$76,000 ─── 売り注文
  ...
$62,000 ─── 買い注文
$60,000 ─── 買い注文
```

価格が $70,000 → $68,000 に下がると買い注文が約定し、$68,000 → $70,000 に戻ると売り注文が約定します。この往復1回でグリッド間隔（$2,000）× 数量分の利益が入ります。

### 得意な相場

- レンジ相場（一定範囲で上下）: 利益機会が増える
- トレンド相場（一方向に大きく動く）: 含み損が膨らむ可能性がある

## リスク管理

### 損切り

スタート時の価格から設定割合（デフォルト5%）下落すると、すべての注文をキャンセルして停止します。

### ポジション制限

同時に保有できるポジション数を制限します（デフォルト: 5）。これ以上は新規買い注文を出しません。

### 自動リトライ

APIエラーやネットワーク切断時、最大3回まで自動的に再試行します。

## 注意事項

1. 最初は必ずTestnetでテストしてください
2. 余裕資金の範囲内で使用してください
3. 定期的にログファイルを確認してください
4. 安定したネットワーク環境で実行してください
5. APIキーは他人と共有しないでください
6. 本番環境では少額から始めてください

## トラブルシューティング

### 「設定エラーがあります」と表示される

`.env` ファイルの `BINANCE_API_KEY` と `BINANCE_API_SECRET` が正しく設定されているか確認してください。

### 「APIリクエスト失敗」と表示される

- APIキーに「スポット取引」の権限があるか確認
- IPホワイトリストが設定されているか確認
- Testnetキーと本番キーを混同していないか確認

### 注文が約定しない

- グリッド間隔が広すぎる可能性があります。グリッド数を増やしてください
- 現在価格がグリッド範囲から外れている可能性があります

### ログの確認方法

`logs/grid_bot_YYYYMMDD.log` に全ログが記録されています。
