## Phase 1: 再起動安全性

再起動で資金を失うリスクを排除する。

### 1.1 手数料反映

**問題**: ライブの PnL 計算に手数料が含まれていない。バックテストのみ対応済み。

**やること**:
- `Settings` に `TRADING_FEE_RATE`（デフォルト 0.001）を追加
- `Portfolio.record_trade()` で SELL 時に買い+売り手数料を利益から差し引く
- ステータス表示に累計手数料を追加

**変更ファイル**: `config/settings.py`, `src/portfolio.py`, `src/bot.py`, `tests/test_portfolio.py`

---

### 1.2 停止時のポジション決済

**問題**: 緊急停止はポジションを決済するが、Ctrl+C は未決済のまま終了。

**やること**:
- `stop()` に `_close_open_positions()` を統合
- `CLOSE_ON_STOP=true/false` で制御（デフォルト true）

**変更ファイル**: `src/bot.py`, `config/settings.py`

---

### 1.3 状態永続化（SQLite）

**問題**: クラッシュ・再起動で全状態が消失。

**やること**:
- SQLite にトレード履歴・グリッド状態・ポートフォリオ統計を保存
- `Portfolio.record_trade()` で DB に INSERT
- `GridStrategy` のグリッド状態（`position_filled`, `order_id` 等）を保存
- 起動時に DB から状態を復元

**新規ファイル**: `src/persistence.py`, `tests/test_persistence.py`
**変更ファイル**: `src/portfolio.py`, `src/bot.py`

---

### 1.4 注文同期

**問題**: 再起動時に取引所のオープン注文と内部状態が不一致になり、重複注文が発生。

**やること**:
- `OrderManager.sync_with_exchange()` を追加
- 取引所のオープン注文と内部 `_active_orders` を突合
- GridStrategy の `position_filled` を実際の約定状況に補正
- `bot.py` の `__init__()` で同期を実行

**変更ファイル**: `src/order_manager.py`, `src/bot.py`
**新規ファイル**: `tests/test_order_sync.py`

---

## Phase 2: 運用品質

### 2.1 Docker 化

---

### 2.2 CI + Lint

**やること**:
- `pyproject.toml` に Ruff / pytest-cov 設定
- GitHub Actions: push/PR でテスト + リント自動実行
- テストカバレッジ 80% 以上を目標

**新規ファイル**: `.github/workflows/ci.yml`, `pyproject.toml`

---

### 2.3 トレード履歴エクスポート

**やること**:
- `Portfolio.export_trades(filepath, format="csv")` を追加
- 停止時に自動で CSV を出力（`EXPORT_ON_STOP=true`）

**変更ファイル**: `src/portfolio.py`, `src/bot.py`, `config/settings.py`

---

### 2.4 WebSocket 対応

**問題**: REST ポーリング（10秒間隔）は API 負荷が高く、約定検知が遅い。

**やること**:
- Binance UserData Stream で注文更新をリアルタイム受信
- MiniTicker Stream で価格をリアルタイム受信
- WebSocket 断絶時は REST フォールバック

**依存**: `websocket-client` 追加

**新規ファイル**: `src/ws_client.py`
**変更ファイル**: `src/order_manager.py`, `src/bot.py`, `requirements.txt`

---

### 2.5 テスト強化

**やること**:
- `GridBot` の統合テスト（モッククライアントでティックシミュレーション）
- `BinanceClient` の異常系テスト（タイムアウト・レートリミット・4xx/5xx）
- `OrderManager` の異常系テスト追加

**新規ファイル**: `tests/test_bot.py`, `tests/test_binance_client.py`
**変更ファイル**: `tests/test_order_manager.py`
